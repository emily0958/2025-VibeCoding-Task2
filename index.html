前端開發者個人網站主視覺區（Hero 區占位）

這裡將放上一句話的自我介紹與核心職稱，例如：「專注於使用者體驗與前端架構的前端工程師」。

目前這裡是 Hero 區的占位內容。未來會放上更精準的品牌標語、簡短的自我介紹，以及引導訪客瀏覽作品集或下載履歷的行動呼籲按鈕，讓整體第一印象更聚焦。

關於我

關於我區塊（之後會用 Section SDD 替換）

這裡目前是 About 區的占位內容，未來會介紹前端工程師的背景、學經歷、職涯轉折與個人價值觀，讓訪客快速理解我是誰、擅長什麼，以及我正在尋找什麼樣的合作或職務機會。

之後會用 Section SDD 的 Prompt 產生新的 section，請開發者把新的 <section>...</section> 程式碼整段貼在 SECTION:ABOUT-START 和 SECTION:ABOUT-END 兩個註解中間，完全取代現在這一區。

開發者提示：若之後需要更進階的排版（例如左右雙欄、時間軸或卡片式簡介），可以在 CSS:ABOUT 區塊覆寫 .section-about 相關樣式，而不需要動到其他區域。

技能專長

技能區塊（之後會用 Section SDD 替換）

這裡目前是 Skills 區的占位內容，未來會整理前端技術、工具鏈與相關軟實力（例如溝通協作、UX 思維等），以標籤、清單或分組卡片的方式呈現，方便閱讀與掃描。

  • 前端技術技能占位（HTML / CSS / JavaScript / RWD 等）
  • 前端框架與工具占位（如 React、版控、編譯打包工具等）
  • 合作與流程占位（跨部門溝通、與設計師協作、敏捷開發等）

之後會用 Section SDD 的 Prompt 產生新的 Skills section，請開發者把新的 <section>...</section> 程式碼整段貼在 SECTION:SKILLS-START 和 SECTION:SKILLS-END 兩個註解中間，完全取代現在這一區。

開發者提示：若未來需要更多視覺層次（例如技能等級、標籤顏色或網格布局），可以在 CSS:SKILLS 區塊中新增或覆寫 .section-skills、.skill-list 與 .skill-pill 的樣式。

作品集

作品集區塊（之後會用 Section SDD 替換）

這裡目前是 Projects 區的占位內容，未來會放上代表性的前端專案，例如個人 Side Project、團隊開發作品或學習成果，並簡短描述專案目標、負責項目與使用技術。

作品卡片占位（一）

這裡是第一個作品卡片的占位文字，未來可以替換成實際專案名稱與說明，例如 RWD 個人網站、互動式元件練習或前後端整合小專案。

開發者提示：正式專案內容產生後，請直接覆蓋整個 <article class="project-card">...</article> 結構。

作品卡片占位（二）

這裡是第二個作品卡片的占位文字，未來可用來展示實際商業專案、團隊合作案例，或與 UX / UI、資料可視化相關的作品。

之後會用 Section SDD 的 Prompt 產生新的 Projects section，請開發者把新的 <section>...</section> 程式碼整段貼在 SECTION:PROJECTS-START 和 SECTION:PROJECTS-END 兩個註解中間，完全取代現在這一區。

開發者提示：如需調整卡片數量、欄位數或 hover 效果，可集中在 CSS:PROJECTS 區塊調整 .project-grid 與 .project-card 的樣式設定。
